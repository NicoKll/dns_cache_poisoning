DNS Cache Poisoning

Domain Name System (DNS):

Um eine Verbindung mit einem am Internet angeschlossenen Gerät aufzubauen, wird eine eindeutige IP-Adressen verwendet. Da für Menschen diese numerischen IP-Adressen zu umständlich und schwierig zu merken sind, hat man Domains wie beispielsweise www.google.com eingeführt. Mit diesen kann ein Benutzer z.B. in einem Webbrowser eine Verbindung mit einem Server aufbauen.

Für einen Computer ist es allerdings viel einfacher mit numerischen Daten zu arbeiten und so muss schlussendlich für eine erfolgreiche Verbindung z.B. mit einer Website die jeweilige Domain in eine IP-Adresse übersetzt werden. Diese Übersetzungen werden im „Telefonbuch des Internets“ oder Domain Name System (DNS) nachgeschlagen. Für die Namensauflösung sind mehrere Nameserver zuständig.


Übersetzung einer Domain

Wenn eine Domain in eine IP-Adresse übersetzt werden sollte, wird als erstes eine Anfrage, ein sogenannter DNS-request, an den DNS-Resolver geschickt. Falls diese Domain kurz zuvor schon einmal übersetzt wurde, hat der DNS-Resolver die passende IP-Adresse in seinem Cache abgespeichert und kann diese dann auch sofort zurückschicken.

Im anderen Fall sendet der DNS-Resolver eine DNS-request an einen sogenannten DNS-Root-Server. Dieser leitet den DNS-Resolver basierend auf der Top Level Domain (.de, .com., .org etc.), kurz TLD, an einen TLD-Nameserver. Dieser wiederum leitet den DNS-Resolver an einen Autoritativen Nameserver. In diesem Nameserver steht dann die Übersetzung der Domain in die IP-Adresse.

Der ganze oben beschriebene Prozess des Nachschlagens braucht natürlich Zeit und aus diesem Grund verwendet der DNS-Resolver einen Cache, um bei einem erneuten Aufruf einer Domain Zeit zu sparen.[03]


DNS Cache Poisoning

Falls es einem Angreifer gelingt, einen falschen Eintrag in den Cache des DNS-Resolvers einzubauen, spricht man von DNS Cache Poisoning. Das führt dazu, dass der DNS-Resolver eine falsche IP-Adresse zurückgibt, die in einem Angriffsfall dem Angreifer gehört. Dies kann dann bedeuten, dass wenn man z.B. www.google.com in den Browser eingibt, nicht auf den Google Server kommt, sondern auf einen anderen Server und so auf eine andere Website. Diese Website kann frei vom Angreifer gestaltet werden, wobei häufig ein Virus oder Computerwurm des Angreifers auf den Computer des Opfers geladen wird.

Auch kann DNS Cache Poisoning für Phishing verwendet werden. Dabei wird das Opfer auf eine Website weitergeleitet, welche die Echte imitiert. So erhofft sich der Angreifer, an Bankinformationen oder Kreditkartenformationen zu kommen. In den meisten Fällen merkt der User, solange die böswillige Seite echt aussieht, gar nicht, dass er sich nicht auf der echten Website befindet, da im Browser die echte Domain steht. [03]

Die Chinesische Firewall verwendet DNS Cache Poisoning in einem grösseren Umfang. Die Cashes der DNS-Resolver in China haben absichtlich falsche Einträge drin, um Websites wie www.twitter.com in ganz China zu blockieren. [04]
